# DESIGN.md — Polyrus 아키텍처 & 전략

## 1. 핵심 가치

- **진실 보정** — 말투/확신도가 아니라 사안의 진실값에 따라 반박.
- **패스 금지 완료** — 회피·중도포기·조기완료·할루시네이션을 시스템 차원에서 금지.
- **원리** — 신뢰성은 베이스 모델을 더 똑똑하게 만들어서가 아니라 *베이스 위 하니스의 구조*에서 나온다.
  다양화 k회 + 검증기 → 시스템 성공률 1 − (1 − p)^k.

## 2. 추론 코어 (문어)

병렬 추론 팔 N개(상관관계 분리) + 콜드스타트 팔(맥락 무시 재유도). 얇은 조정 뇌는
이견을 보존하고 보정된 불확실성을 보고한다(조기 단일 선택 금지). 콜드 사슬과 워밍 사슬이
갈리면 위험 신호.

## 3. 패스 금지 레이어 (M1–M6)

- **M1 완료 원장** — 하위 목표 분해 + DoD + 검증기. 전 항목 검증-완료/에스컬레이션 전 종료 불가.
- **M2 무회피 심판** — 출력을 수행 vs 회피로 분류, 회피 시 재시도 강제.
- **M3 에스컬레이션** — 미완료의 유일한 합법 경로(구체 질문 동반). 조용한 패스 금지.
- **M4 다양화 재시도** — 실패 시 다른 접근으로 k회 후에만 에스컬레이션.
- **M5 진실 게이팅** — 완료는 모델 주장이 아니라 검증기 판정.
- **M6 출처 원장** — 모든 결론에 증거 사슬, 무근거 플래그.

## 4. 검증 뱅크 (까마귀) — 해자

4티어. 시스템은 각 주장이 어느 티어인지 알아야 하고, 낮은 티어에 높은 티어 확신을 붙이지 않는다.

- **T1 실행 진실(강)** — 컴파일·테스트·린트·보안·차등. 결정적 무-LLM.
  굿하트 방어: 뮤테이션 테스팅, 프로퍼티 기반, DoD-우선, 홀드아웃.
- **T2 교차검산(중)** — 콜드스타트 팔 독립 재구현 ↔ 차등 테스트.
- **T3 출처대조(중)** — API 존재를 패키지 메타데이터에 대조(환각 차단).
- **T4 적대비평(약)** — 레드팀 + 퍼징. "생존 + 우려"로 보정.

원칙: ① 독립성(생성기와 같은 식으로 틀리는 검증기는 무용) ② 굿하트 저항 ③ 검증기 자체 보정
(확신도 = 검증기 신뢰도). 해자 = 도메인별 검증기 라이브러리(코드·재무·검색).

**정직한 경계** — 검증기는 "스펙 대비 정확성"을 잠그지 "스펙의 정당성"은 못 가린다.
스펙이 쓰레기면 "검증된 채로 틀린" 코드가 나온다 → DoD 생성기가 정당성의 첫 방어선.

## 5. 기술 구현 (경쟁자에서 배운 것)

OpenClo·Hermes는 같은 골격: **로컬 게이트웨이 + LLM+도구 에이전트 루프 + 플러그형 스킬/메모리 + 모델 비종속.**

- Hermes(Python, MIT): `run_agent.py` 핵심 루프, SQLite FTS5 메모리, 플러그형 ABC(메모리/컨텍스트),
  ThreadPoolExecutor 병렬 도구, 프로바이더 폴백 체인, OpenAI 호환. 자기진화는 "변이는 pytest 100% + PR 리뷰" 게이팅.
- OpenClaw(MIT): 중앙 Gateway(WS+HTTP 멀티플렉싱), 멀티턴 루프(텍스트-only 응답 시 종료),
  하트비트 데몬(HEARTBEAT.md), exec 승인 정책(끄면 무단 실행 → 책임 리스크).

**그들의 루프는 모델이 완료를 선언하면 종료된다.** Polyrus는 같은 기판을 쓰되 종료 조건을 뒤집어
*검증기 합치에서만 종료*한다.

- **재사용** — MCP 도구, 샌드박스 백엔드(Docker/subprocess, Hermes식 다중 백엔드),
  SQLite 원장/메모리, 병렬 실행(ThreadPoolExecutor), 모델 비종속(OpenAI 호환 + 어댑터 + 폴백).
- **빌드 형태** — (a) 독립 하니스, (b) 기존 에이전트(Hermes/OpenClaw/Claude Code) 위
  검증-게이팅 래퍼/게이트웨이. "싸우지 말고 감싸라."

## 6. 플랫폼 & 수익화

경쟁자는 MIT·무료·셀프호스팅, 약한 수익화(Hermes=Nous Portal 구독, OpenClaw=관리형 호스팅).
Polyrus는 신뢰성/거버넌스/검증을 유료화한다.

- **오픈코어** — OSS 하니스 + 검증기 SDK(채택 플라이휠).
- **관리형 플레인** — 스케일 검증 클라우드, 큐레이션 검증기 뱅크, 홀드아웃 인프라,
  감사·승인(증거 게이트 = 컴플라이언스), 완료 SLA.
- **검증 게이트웨이** — 아무 에이전트를 가리키면 검증-게이팅 + 감사를 입히는 OpenAI 호환/MCP 엔드포인트.
- **검증기 마켓플레이스** — 도메인 검증기 수수료(복리 해자).

**파는 단위** — "검증된·비조작·완료된 태스크"(태스크당 과금) + 기업 거버넌스 시트.

## v0.4 추가 개념 (기획안 동기화)

- **두 모드 (수렴/발산).** 코드·사실 = 수렴(검증기 왕, 낮은 온도, 하나의 검증된 답).
  네이밍·각도·카피 = 발산(폭 넓힘, 팔 다양, 클리셰 거부, 취향 판정, 스프레드 보존).
  창의성 상실 = 영합과 같은 뿌리(모드 붕괴) → 패스 금지의 창의 버전 = '뻔한 답 거부'.
- **가중치 다이얼 A~D** (`A 발산↔수렴 / B 속도↔철저 / C 개입↔자율 / D 신중↔과감`).
  시스템이 작업마다 *추천*하고 사용자가 override. 프리셋(브레인스토밍/출시/리서치)이 표면.
  **페르소나·CLAUDE.md(WHAT, 프롬프트, 지속)와 분리** — 다이얼은 HOW(오케스트레이션 정책,
  하니스가 *집행*, 작업마다 바뀜). 위계는 CSS 캐스케이드(프로젝트 기본값 → 작업 추천 → override).
- **이해 검증 (5.5).** 출력 검증의 상류 — DoD를 이해 산물로, 가정 표면화, 해석/실행 확신도 분리,
  팔의 해석 divergence = 모호성 감지기. '막는 질문' 대신 'X로 읽고 진행 중 — 아니면 고쳐줘'.
- **멈춤·누락 제거.** 검증기를 *루프 종료조건*으로 — 미완 탐지 → 거부 → 빠진 항목만 이어서 →
  재검증 반복. 요구사항은 외부 원장에, 검증은 독립(비-모델 오라클)으로.
- **장시간 자율 (6.4).** 체크포인트·저널링·되돌릴 수 없는 행동 격리(확인 트레이).
- **자격증명 (6.3 / SECURITY.md).** 로컬 우선·비전송, env식 글로벌 계정 + OS 키체인,
  모델 컨텍스트 우회, 전면 리댁션, 사용 시 동의 게이트, 면책. 스텁: `src/polyrus/secrets.py`.

## v0.5 강화 — 해자 재정의 · 신뢰성 증명 · 루프 정지정책

> 출처: `20260618_1116_Polyrus_해자·벤치마크·예산정책_강화_기획안.md`. v0.4의 약한 3축
> (해자가 커머디티 / 신뢰가 증명 안 됨 / 루프 비정지 위험)을 정책·숫자·측정법으로 못박는다.
> 셋은 하나의 루프다 — ③이 루프를 출시가능하게, ①이 데이터를 복리화, ②가 그 복리를 측정.

### 7. 해자 재정의 — 검증기가 아니라 *보정된 증명*이 해자

**정직한 인정.** T1 실행 진실(컴파일러·pytest·mutmut·린터)은 전부 OSS 커머디티. "도메인 검증기
라이브러리 = 해자"는 절반만 맞다. 라이브러리는 복제된다. 복제 안 되는 건 따로다.

- **(진짜 해자) 보정 코퍼스 = 데이터 플라이휠.** No-Pass 루프를 돌리는 자만 라벨 한 점을 쌓는다:
  `(태스크, 모델_주장, 검증기_판정, 다운스트림_결과/사람_override)`. 이 코퍼스 → 도메인별 검증기
  신뢰도 곡선 → 보정된 확신도. 코드 해자가 아니라 **데이터 네트워크 효과**.
  플라이휠: `태스크↑ → 보정↑ → 확신 신뢰↑ → 유저↑ → 태스크↑`.
  **OSS 코어를 푸는 건 해자를 거저 주는 게 아니라 보정 플레인으로 들어오는 고객 획득 깔때기.**
- **(보조 해자 A) 검증기 조합·오케스트레이션 정책.** 언제 어느 티어를 부르고, T1–T4 판정을 어떻게
  하나의 보정 확신도로 합치고, 팔 divergence를 언제 모호성으로 읽을지 — 코퍼스에 튜닝돼 복제 불가.
- **(보조 해자 B) 검증기 마켓플레이스 = 양면 네트워크 효과.** 도메인 전문가 게시 ↔ 유저 소비,
  그리고 마켓플레이스 검증기도 보정 코퍼스를 먹인다(배포이자 해자 사료).
- **플랫폼 흡수 대응.** ① 모델 비종속 = 구조적 反흡수(벤더의 검증-완료는 *자기 모델 위에서만* 돈다;
  교차모델·교차에이전트 거버넌스는 벤더가 자기 락인 깎기 싫어 안 만든다). ② 보정 코퍼스는
  벤더 중립·교차 도메인. ③ 컴플라이언스 = **직무 분리(separation of duties)** 상품 — 감사인은
  "모델이 자기 숙제 자기 채점"을 안 믿는다. *모델 벤더가 아니라서 더 신뢰받는다.*

> **새 해자 한 줄.** 해자는 검증기가 아니라, 검증기들의 *보정된·교차벤더·감사가능한 증명
> (attestation)* 과 그것을 복리로 키우는 *보정 코퍼스*다.

### 8. 신뢰성 증명 — Polyrus 신뢰성 벤치마크(PRB)

기존 벤치(SWE-bench·τ-bench·GAIA)는 *능력*을 잰다. 우리는 *정직성(failure honesty)* 을 잰다 —
정직한 답이 "못 한다/모호하다/이 항목이 빠졌다"인 함정을 일부러 심는다. (`examples/prb/` 시드)

- **프로브 패밀리.** 불가능·과소명세(→에스컬레이션) / 부분완료 트랩(→빠진 항목 명시) /
  영합 프로브(→진실대로 반박) / 환각 미끼(→존재 안 함 플래그) / 책임전가 프로브(→그냥 수행).
  각자 *정답(ground-truth honest action)* 라벨링.
- **측정법 (KPI를 ↑↓에서 숫자로).**
  - 보정 오차 → **ECE(Expected Calibration Error) + Brier score**. 닭-달걀 해소: 정답은
    *홀드아웃 평가셋*(도메인당 수백 건 라벨, 앵커)에만, 프로덕션 코퍼스는 거기 대조해 검증.
  - 에스컬레이션 적정성 → 불가능-부분집합에서 **precision/recall**(전부 에스컬레이션하는 가짜
    No-Pass를 precision이 잡는다).
  - 검증기 신뢰도 → 검증기별 FP/FN을 홀드아웃에서 시계열(보정 코퍼스를 읽을 수 있게).
- **벤치마크 굿하트 자기적용.** 공개 train split + 비공개 holdout(분기 갱신), 헤드라인은 항상 홀드아웃.
- **8 ↔ 7 동치.** 홀드아웃 ECE 하락 = 보정 코퍼스 플라이휠이 도는 가시적 증거.

### 9. 루프 정지 / 예산 정책 — No-Pass ≠ No-Stop

`harness.py` 루프 불변식의 정지 보장. "절대 포기 금지"는 "절대 멈춤 금지"가 아니라 **No-Silent-Stop**.

- **루프는 *항상* 종료한다** — 셋 중 하나의 합법 상태로: ① 검증-완료, ② 에스컬레이션(구체 질문+부분
  결과), ③ **예산경계 에스컬레이션**(예산 소진 → 강제 M3, 검증된 부분 + 남은 미검증 항목 명시).
  불법(금지) 상태: 조용한 가짜-완료/조용한 누락/확신에 찬 오답. **천장 도달 = 합법 M3 변환.**
- **예산 객체(1급 자원).** 루프 시작 전 `budget envelope` 부여(다이얼 B에서 파생):
  `max_tokens`($), `max_arms` N, `max_retries` k, `wall_clock`. 6.1 게이트의 "비용 X"가 실제 숫자.
  → `src/polyrus/budget.py`(신규), `harness.py`의 `adaptive_k`가 여기서 상한을 읽음.
- **적응형 깊이 정량화.** ① **막힘 감지**: k 시도가 같은 하위목표·같은 검증기를 같은 방식으로 실패
  (팔 다양성 붕괴 = 새 정보 없음) → k+1 태우지 말고 에스컬레이션. ② **비용순 티어 사다리**:
  T1(싸고 결정적) 먼저 → 모호/저확신일 때만 T2/T3 → T4는 고위험·실행불가 주장에만.
  ③ **확신도 목표 정지**: `(검증완료 ∧ 목표확신) ∨ 예산소진 ∨ 막힘감지` 중 먼저 오는 것.
- **비정지 보장.** 매 반복이 유한 예산에서 엄격히 차감 + 소진은 결정적으로 에스컬레이션 → 무한 루프 경로 없음.
- **비용 경제학(리스크 9-(1) 진짜 완화).** 5~10배는 *평균이 아니라 꼬리*. 대부분 태스크는 T1·단일
  팔에서 단락(≈1~1.5배); 고위험만 팬아웃(드물고, 게이트로 유저 승인). **검증 비용 = 게이트로 승인된
  꼬리에만.** 과금(태스크당)도 오른 티어 사다리 높이에 비례 → 유닛 이코노믹스가 예산 객체에서 떨어짐.

### 10. 배포·폼팩터 — Wrap-First (own-later)

> 출처: `20260618_1141_Polyrus_배포전략_wrap-first_기획안.md`. 폼팩터가 아키텍처를 강제하므로
> 구현 *전에* 못박는다. **첫 웨지 = Claude Code Stop-hook 래퍼(확정).**

**거르는 필터.** 핵심 불변식("완료=검증기 판정, 루프가 *집행*")은 제어흐름 속성. 그래서 폼팩터를
거르는 질문은 하나 — *그 표면이 종료조건을 집행하나, 모델에게 부탁만 하나?* MCP 도구는 모델이
무시 가능(부탁) → 약한 형태. 게이트웨이는 출력만 게이팅(중간). 집행 가능한 건 **독립 CLI 루프**와
**Stop-hook 래퍼** 둘. 마찰 최저 + 교차벤더 + 꼬리비용만인 후자가 웨지.

- **왜 Stop-hook 래퍼.** Claude Code `Stop` 훅이 `block + reason` 반환 → 모델 못 멈추게 + reason
  재주입. 이게 종료조건 역전 그대로: 모델 "끝" → 완료 원장 vs 검증기 뱅크 → 미검증이면
  `block: "빠진 항목 X"` → 합치/예산경계까지. 5.4 흡수방지·5.7 꼬리비용·"싸우지 말고 감싸라"가 한 점.
- **빌드 원칙 6.** ① 코어=앱 아니라 임베더블 검증 엔진(I/O 가정 0). ② 모델/에이전트 비종속 경계를
  인터페이스(`ModelClient`·`AgentAdapter`)로 — 흡수방지의 코드적 실체. ③ 첫 어댑터=Stop-hook 래퍼
  (독립 에이전트 안 짬). ④ 로컬/관리 분리선을 VerifierBank에 day-1(`runs_local|runs_managed`,
  리댁션 경계=무료→유료 seam). ⑤ 보정 코퍼스 emit을 엔진 1급 출력으로(Phase 0부터). ⑥ 게이트웨이는
  나중 어댑터(엔터프라이즈), 웨지 아님.
- **어댑터 아키텍처.** 한 코어, 여러 표면. 코어는 어댑터를 모른다 — 어댑터가 `(태스크,DoD,예산)`을 주고
  `(verdict,확신도,코퍼스레코드)`를 받음. graduate: 래퍼(집행) → CLI(집행+문어 병렬팔/발산) →
  게이트웨이(범용·거버넌스). 같은 코어가 셋을 떠받침.
- **패키징.** `pipx install polyrus`(CLI)+`pip install polyrus`(SDK)+MIT 검증기 SDK = 채택 플라이휠.
  래퍼는 OSS 패키지 포함(`polyrus wrap claude`). 관리형 플레인=SaaS(검증 태스크당+거버넌스 시트).
