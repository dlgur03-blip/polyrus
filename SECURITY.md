# 보안 · 자격증명 · 면책 (기획안 6.3)

## 로컬 우선 원칙
- 자격증명(비밀)은 **사용자 머신에만** 존재합니다. Polyrus 서버, 관리형 검증 플레인,
  텔레메트리로 **절대 전송되지 않습니다**.
- 비밀값은 OS 비밀 저장소(macOS Keychain / Windows Credential Manager /
  Linux Secret Service)에 두고, `account.env`는 *참조만* 담습니다.
- 평문 `.env`는 폴백입니다 — 파일 권한 `600`, 반드시 `.gitignore`.
- 비밀은 **모델 프롬프트에 들어가지 않습니다.** 키체인에서 도구 실행 경계
  (예: HTTP 인증 헤더)로만 주입되어, 모델과 모델 제공자에게도 노출되지 않습니다.
- 모든 로그·완료 원장·출처 원장·오류 메시지는 `redact()`로 마스킹된 값만 봅니다.

## 자격증명 사용 = 동의 게이트
- 자격증명으로 부수효과 행동(전송·결제·삭제)을 할 때는 증거조사 게이트(6.1)와
  행동 안전(6.2)을 그대로 상속합니다 — 읽기는 낮게, 쓰기는 미리보기 + 동의.
- 되돌릴 수 없는 행동은 끄지 못하는 게이트로, 단 원탭 미리보기로.

## 면책 (Disclaimer)
에이전트는 사용자의 자격증명으로 **사용자의 지시·책임 하에** 행동합니다.
무엇을 인가할지는 사용자의 책임이며, 되돌릴 수 없는 행동은 게이트로 승인합니다.
Polyrus는 비밀을 로컬에 보관하고 전송하지 않으나, **로컬 머신 자체의 침해까지는
보증하지 않습니다**(무보증, as-is). 설정 시점에 이 고지를 명시적으로 표시합니다.
